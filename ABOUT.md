# About JD Flow

JD Flow was created from a real operational need.

The dispatch process between Tijuana and San Diego required better visibility, document control and status tracking. Before JD Flow, much of the process depended on manual communication, emails, file handling and individual follow-up.

This project was created to centralize the dispatch workflow and make operational work easier, faster and more traceable.

JD Flow is not only a software project. It is a process improvement initiative built from daily operational experience.

## Vision

To transform dispatch coordination into a measurable, visible and technology-driven workflow.

## Main Objective

Reduce manual work, increase visibility and give both Tijuana and San Diego teams a single place to track dispatch progress.
