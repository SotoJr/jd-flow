# JD Flow

JD Flow is an internal workflow platform created to improve dispatch visibility, document control and operational traceability between Tijuana and San Diego warehouse operations.

## Purpose

The goal of JD Flow is to reduce manual follow-up, eliminate unnecessary email dependency, improve document visibility and provide a clearer operational status for dispatch orders.

## Core Modules

- Dispatch Board
- Document Package Upload
- Status Tracking
- User Roles
- Audit History
- Operational KPIs

## Current Status

JD Flow is currently functional as an internal operational tool.

## Tech Stack

- Hostinger Horizon
- Supabase
- SILA integration
- Tampermonkey connector
- JavaScript
- Web application frontend
