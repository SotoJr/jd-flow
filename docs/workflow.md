# JD Flow Workflow

## Current Operational Flow

1. Client creates or uploads order information in SILA.
2. Tijuana reviews the order and documents.
3. Order is sent to JD Flow Dispatch Board.
4. Final signed package is uploaded.
5. San Diego downloads and prints the package.
6. Order is dispatched.
7. Status and history remain visible in JD Flow.

## Main Statuses

- GENERADO
- ENVIADO_SD
- IMPRESO
- DESPACHADO
- DETENIDO

## Objective

Provide a clear operational board where users can understand what is pending, what has been reviewed, what has been printed and what has been dispatched.
